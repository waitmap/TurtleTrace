## 1. 需求背景

### 1.1 用户画像

持有**低波动蓝筹股**（中国神华、长江电力等）的个人投资者，操作风格为波段交易——低位买入、高位卖出。

### 1.2 核心痛点

卖出后陷入两难：
- **怕踏空**：卖了它继续涨，想买回来
- **怕追高**：现在买回来又怕买在高位被套

### 1.3 解决方案

**安全垫 + 金字塔分批回购法：**
- 已实现的利润作为安全垫，提供容错空间
- 分 3 批（30%/40%/30%）在价格回调到不同位置时买入
- MA60 作为强支撑参考，跌破 MA60 触发跳级买入

---

## 2. 核心概念

### 2.1 安全垫（Safety Cushion）

```
安全垫 = 已实现净利润 / 总回购预算 × 100%
```

安全垫表示：即使回购后股价继续下跌，你最多能承受多少百分比的浮亏而不伤及本金。

### 2.2 金字塔分批回购

| 批次 | 触发条件 | 买入比例 | 说明 |
|------|---------|---------|------|
| 第1批 | 距卖出价回撤 ≥ **3%** | **30%** 预算 | 小幅回调，低风险 |
| 第2批 | 距卖出价回撤 ≥ **6%** | **40%** 预算 | 中等回调，加大仓位 |
| 第3批 | 价格触及 **MA60**（±1%） | **30%** 预算 | 强支撑位，打满 |

### 2.3 跳级满仓机制

如果价格**直接跳空跌破 MA60**（比如 -8% 砸穿），说明前两批都没机会执行，此时：

- **跳过前两批**
- **一次性买入预算的 70%**
- 剩余 30% 等待继续跌 3% 后再补

### 3.1 修改 `src/types/index.ts`

```typescript
export interface Position {
  // ... 现有字段不变 ...

  // 新增：
  rebuyPlan?: RebuyPlan
  realizedProfit?: number     // 该股已实现净利润（自动计算）
}

export interface RebuyPlan {
  totalBudget: number         // 计划用于该股回购的总本金预算
  manualMa60?: number         // 手动输入的 MA60 价格（可选，覆盖自动值）
  batchesExecuted: number     // 已执行的回购批次数量 (0/1/2/3)
  enabled: boolean            // 是否启用该股的回购计划
}
```

  }
}

## 4. 计算逻辑

### 4.1 输入参数

| 参数 | 来源 | 说明 |
|------|------|------|
| `currentPrice` | 东方财富 API（已有） | 当前实时价 |
| `lastSellPrice` | 交易记录自动计算 | 最近一次卖出的加权均价 |
| `ma60Price` | API 自动拉取 + 手动覆盖 | MA60 均线价 |
| `realizedProfit` | 交易记录 FIFO 计算 | 该股已实现净利润 |
| `totalBudget` | 用户配置 | 回购总预算 |
| `batchesExecuted` | 用户配置（自动更新） | 已执行批次 |

### 4.2 计算步骤

```
1. dropPercent = (lastSellPrice - currentPrice) / lastSellPrice
2. distanceToMa60 = (currentPrice - ma60Price) / currentPrice
3. safetyCushion = realizedProfit / totalBudget × 100%

4. 状态判断（优先级从高到低）:

   4a. 防跳空：
       IF currentPrice <= ma60Price × 1.01 AND batchesExecuted < 2
         状态 = '跳级满仓'
         买入金额 = totalBudget × 0.7
         新 batchesExecuted = 2

   4b. else IF dropPercent < 0.03 OR batchesExecuted >= 3
         状态 = '观望'
         买入金额 = 0

   4c. else IF dropPercent >= 0.03 AND batchesExecuted === 0
         状态 = '第一批回购（30%）'
         买入金额 = totalBudget × 0.3
         新 batchesExecuted = 1

   4d. else IF dropPercent >= 0.06 AND batchesExecuted === 1
         状态 = '第二批回购（40%）'
         买入金额 = totalBudget × 0.4
         新 batchesExecuted = 2

   4e. else IF currentPrice <= ma60Price × 1.01 AND batchesExecuted === 2
         状态 = '第三批回购（30% 买满）'
         买入金额 = totalBudget × 0.3
         新 batchesExecuted = 3

5. 买入股数 = floor(买入金额 / currentPrice / 100) × 100
```

### 4.3 输出格式

```typescript
interface RebuyAdvice {
  status: '观望' | '第一批回购' | '第二批回购' | '第三批回购' | '跳级满仓'
  statusColor: 'gray' | 'yellow' | 'blue' | 'green' | 'red'
  dropPercent: number
  distanceToMa60: number
  safetyCushion: number
  batchAmount: number
  batchShares: number
  suggestPrice: number
  totalBudgetLeft: number
  summary: string
}
```

## 8. 开发步骤

### Step 1: 获取源码

```bash
git clone https://github.com/TurtleTrace2026/TurtleTrace.git
cd TurtleTrace
npm install
```

### Step 2: 扩展类型

编辑 `src/types/index.ts`，添加 `RebuyPlan` 接口，在 `Position` 中添加 `rebuyPlan?` 和 `realizedProfit?` 字段。

### Step 3: 创建 `rebuyService.ts`

纯函数，包含 `calculateRebuyAdvice`、`getLastSellPrice`、`calculateRealizedProfit`。

### Step 4: 创建 `rebuyStorageService.ts`

localStorage CRUD，参考 `tCalculatorService.ts` 的 pattern。

### Step 5: 修改 `accountService.ts`

在加载持仓数据时自动计算 `realizedProfit` 并注入到 `Position` 对象。

### Step 6: 创建 `RebuyDashboard.tsx`

完整的页面组件，包含卡片列表、配置弹窗、状态色块。

### Step 7: 修改 `App.tsx`

- 引入 `Target` 图标
- 在 `tabs` 数组加入 `rebuy` tab
- 在渲染区加入 `RebuyDashboard`

### Step 8: 验证

```bash
npm run dev        # 开发模式
npm run build      # 编译检查
```

## 9. 部署指南

### 9.1 环境要求

- Node.js ≥ 18（LTS）
- npm ≥ 10

### 9.2 首次运行

```bash
# 1. 下载代码
git clone https://github.com/TurtleTrace2026/TurtleTrace.git
cd TurtleTrace

# 2. 安装依赖
npm install

# 3. 启动开发服务器
npm run dev
# 浏览器打开 http://localhost:5173
```

### 9.3 打包部署

```bash
# 编译为静态文件
npm run build

# 产物在 dist/ 目录，可直接用浏览器打开
# 或使用预览服务器
npm run preview
# 浏览器打开 http://localhost:4173
```
